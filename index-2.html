開発現場の課題とAIによる解決

「単純なコーディング作業に時間を取られ、新しい技術の学習や設計の深掘りができない…」

「テストコードを書く時間が足りず、品質に不安を抱えたままリリースすることがある…」

「レガシーコードの解読や修正に膨大な時間がかかり、モダナイゼーションが進まない…」

「コードレビューに時間がかかりすぎたり、レビュアーによって指摘の質にばらつきがある…」

「AIによる開発支援ツールは色々あるけど、どれをどう使えば本当に効果が出るのかわからない…」

開発工程の課題

コーディングの非効率、テスト工数の肥大化、品質確保の難しさ、レガシーシステムの保守負担、デバッグの長期化。これらは開発者のモチベーション低下やプロジェクトの遅延、品質低下に繋がります。

AIによる解決策

最新AIツールを活用し、コード生成、リファクタリング、テスト自動化、レビュー支援、デバッグ効率化を実現。開発プロセス全体の生産性と品質を飛躍的に向上させ、エンジニアがより創造的な業務に集中できる環境を作ります。

対象者

ソフトウェアエンジニア（全レベル）、テックリード、QAエンジニア、テストエンジニア、DevOpsエンジニア、SRE、AI活用による開発プロセス改善を目指す全ての方

研修に申し込む

研修カリキュラム（開発工程編）

本研修は、ソフトウェア開発におけるコーディング、リファクタリング、テスト、デバッグ、CI/CD連携といった中核工程において、生成AIを駆使して生産性と品質を劇的に向上させるための実践的なスキルを習得することを目的とします。

研修日数/時間（例）：2日間（計14時間）

Day 1: AIによる革新的コーディングと実装効率化

モジュール1: AI駆動開発概論（開発工程編）と主要AIコーディングツール詳解 (2.5時間)
  • 開発工程におけるAI活用の最前線
    • AIアシステッド開発からAIドリブン開発への進化（開発工程における具体例）
    • 期待される効果：開発スピード向上、リソース効率化
    • AI駆動開発が適している領域（定型業務、レガシー、プロトタイピング、テストコード生成）
  • 主要AIコーディングツールの徹底比較と実践セットアップ
    • GitHub Copilot: 機能、強み、対応IDE、MCP連携
    • Cursor: 特徴、強み（プロジェクト全体インデックス化、推論速度、レガシーコード対応）
    • Claude/Clain (VSCode拡張): LLM選択自由度、エンジニア満足度、生産性効果事例
    • Windows Copilot (コーディングエージェント): コード間関連性の意味空間理解
    • Devin (自律型AIエンジニア): 概要、リモートインスタンスでの並列稼働、CIでの実装自動化事例
  • 【演習1-1】各ツールのインストール、基本設定、簡単なコード生成タスクでの操作感比較
モジュール2: AIアシステッド・コーディング実践とプロンプト術 (2.5時間)
  • 開発工程特化型プロンプトエンジニアリング
    • コーディング支援のための具体的指示方法（機能仕様、データ構造、期待する出力形式の明示）
    • リファクタリング、バグ修正、アルゴリズム提案依頼のコツ
    • 既存コードをコンテキストとして提供する方法
  • 「バイブコーディング」の効果的な活用法と注意点
    • アイデアとしての初期コード生成、対話を通じた段階的実装、未知技術の試作品作成
    • 「AI酔い運転」問題の回避（コード理解の重要性、影響範囲の把握）
  • AIによるコードコメント・ドキュメント自動生成
    • JSDoc、Python Docstring等の形式に合わせたコメント生成
    • READMEファイルや技術ブログ記事のドラフト作成支援
  • 既存コードベースの理解とAIによるリファクタリング支援
    • 大規模コードベースの解析（大規模トークンモデル連携）
    • OSSや初見ライブラリの理解促進（DeepWiki活用）
  • 【演習2-1】小規模な仕様書に基づき、AIと対話しながら主要機能を実装
  • 【演習2-2】提供された既存コードをAIを用いてリファクタリングし、可読性と保守性を向上
モジュール3: AIエージェントによる実装自動化とレガシーシステム対応 (2.0時間)
  • AIエージェント（Devin等）の活用戦略
    • タスク分割の重要性（「ピット・オブ・デス」回避）
    • CI/CDパイプラインへの組み込みによる実装・修正タスクの自動化
    • コードベースの調査・探索（「壁打ち」相手としての活用）
  • 【演習3-1】簡単な機能追加またはバグ修正タスクを定義し、Devin（またはその概念を模したシミュレーション環境）に依頼し結果を検証
  • AIを活用したコードからのドキュメント生成
    • ドキュメント不足のコードに対するLLMを用いた自動ドキュメント生成
    • Code to Promptテクニック：コードをプロンプトに変換して高品質な説明を生成
    • 大規模コードベースからの技術仕様書・アーキテクチャ図の自動生成
  • 【演習3-2】提供されたコード（一部）をAIで解析し、Code to Promptテクニックを用いて詳細な技術ドキュメントを生成

Day 2: AIによるテスト自動化と品質保証、そして未来へ

モジュール4: AIを活用したテスト戦略とテストコード生成 (2.5時間)
  • AI時代のテスト戦略と人間とAIの役割分担
    • テストピラミッドにおけるAIの適用範囲（ユニットテスト、結合テスト、E2Eテスト）
    • テストのベースケース作成（人間）とバリエーション生成（AI）
  • AIによる各種テストコードの自動生成
    • 主要言語・フレームワークに対応したユニットテストコード生成
    • APIテスト、UIテスト（Selenium, Playwright等）のスクリプト生成支援
  • テストパターンの網羅性向上と品質保証
    • 境界値分析、同値クラステスト、デシジョンテーブルに基づくテストケースのAI提案
    • エラーハンドリング、セキュリティ脆弱性に関するテストケースの洗い出し支援
  • 【演習4-1】Day1で作成したコードに対し、AIを用いてユニットテストコードを生成し、カバレッジを確認
  • 【演習4-2】特定のAPI仕様に基づき、AIを用いてテストケースとテストスクリプト（主要部分）を生成
モジュール5: AIによるコードレビューとCI/CD連携 (2.5時間)
  • AIを活用したコードレビューの革新
    • プルリクエストの自動サマリー作成（変更点の要約、影響範囲の提示）
    • AIによる静的解析、コーディング規約チェック、潜在的なバグやアンチパターンの指摘
    • PRエージェントの活用と人間による本質的なレビューへの集中
  • CI/CDパイプラインへのAIツールの統合
    • 自動テスト実行結果のAIによる分析とレポート生成
    • デプロイ前の最終チェック（セキュリティスキャン、パフォーマンス予測等）へのAI活用
  • 【演習5-1】提供されたプルリクエストに対し、AIレビューツールを使ってレビューコメントを生成・評価
  • 【演習5-2】（概念実習）CI/CDパイプラインの特定ステージにAIによる品質チェック処理を組み込むシナリオを設計
モジュール6: AI時代のデバッグ、運用、そして開発者の進化 (2.0時間)
  • AIによる効率的なデバッグとトラブルシューティング
    • エラーメッセージ、スタックトレースのAIによる解析と原因特定支援
    • バグ修正案のAIによる提示と適用
    • パフォーマンスボトルネックの特定と改善提案
  • AI駆動開発の導入・推進における注意点（開発現場視点）
    • セキュリティ対策（学習データからの情報漏洩リスク、悪意のあるプロンプト対策）
    • コスト管理（API利用料のモニタリング、プロンプトキャッシュの活用）
    • AI出力の品質ばらつきへの対処、レビュー負荷増と認知負荷への対策
  • 今後の展望：AIネイティブな開発ライフサイクルと開発者の役割変化
    • 「コンティニュアス・インテント・レコンシリエーション」の概念
    • DRE（Developer Reliability Engineer）など新しい専門職の可能性
    • AIとの効果的な協業パターン（チームモデル、役割分担の最適化）
    • 継続的学習と「AIとの強心化」の重要性
  • 総括、質疑応答、アクションプラン策定支援

川西智也は、京都大学大学院情報学研究科を修了後、学生時代からKLab社でプログラミングに携わってきました。関西電力に入社してからは、光ファイバー回線の開通や保守運用を支える各種ツールを自ら開発し、部門表彰を受賞。その後はサービス企画やIT企画、技術統括など多彩な役割を経験しました。

2015年、電力自由化の潮流に乗ってENECHANGE社の創業メンバーとして入社。チーフエンジニアとしてサーバサイド開発やインフラ設計、エンジニア採用に従事し、同社のマザーズ上場（2020年）を技術面から力強く支えました。

2021年に独立し、合同会社ロイヤルピースを設立。IT系スタートアップを中心に経営コンサルティングおよびITコンサルティングを提供するとともに、2025年には不動産系スタートアップ・株式会社スペースリーのCAIOに就任しています。

本研修では、氏のエンジニアリングおよびアーキテクトとしての深い知見を活かし、AIを開発工程で最大限に活用するための具体的なテクニック、ツールの選定眼、導入時の注意点を解説します。
